# 작업 기본 규칙

## Git 브랜치 전략
- 기본 개발 브랜치는 `dev`를 사용한다.
- 기능 개발은 항상 `dev`에서 분기한다.
- 기능 브랜치 이름 규칙은 `feat/<기능명>`으로 한다.
- 기능명은 소문자와 하이픈(`-`)을 사용한다. 예: `feat/calendar-redesign`

## 작업 순서
1. `dev` 최신 상태로 이동한다.
2. `feat/<기능명>` 브랜치를 만든다.
3. 기능 구현 후 관련 파일만 스테이징한다.
4. 커밋 메시지는 명확한 타입 접두어를 사용한다.
5. 원격 저장소가 연결되어 있으면 브랜치를 푸시하고 PR을 생성한다.

## 커밋 메시지 규칙
- `feat: 사용자 기능 추가`
- `fix: 버그 수정`
- `refactor: 구조 개선`
- `chore: 설정/빌드/문서 작업`

## 코드 스타일
- 코드 주석은 한국어로 작성한다.
- 변경 이유가 필요한 부분에는 짧고 명확하게 주석을 남긴다.
